# GeoAgent — AI驱动的地理空间分析平台

## 1. 项目愿景

**一句话定位：** 用自然语言操控地图，AI自动完成从数据接入→空间分析→可视化渲染→报告生成的全链路。

**产品名：** GeoAgent  
**Slogan：** "Talk to your map."

---

## 2. 为什么做这个

### 2.1 市场现状

| 现状 | 痛点 |
|------|------|
| GIS分析门槛高 | 用户需要掌握ArcGIS/QGIS等专业工具，学习成本极高 |
| AI+GIS停留在论文阶段 | 大量研究论文探讨可行性，但没有可用的开源产品 |
| 现有GIS平台不智能 | SuperMap/ArcGIS Online的AI功能仅限基础图像识别 |
| 自然语言+地图无成熟方案 | 搜"GIS Copilot"、"Map AI Assistant"，市面上几乎为零 |

### 2.2 我们的机会

- LLM（大语言模型）已经成熟到可以理解复杂空间查询指令
- LangChain/LangGraph提供了完整的Agent工具链
- Cesium/MapLibre等前端引擎可以实现高质量的交互式可视化
- **没有人把这些串起来做成一个产品**

## 3. 核心场景

### 场景1：智能空间查询
```
用户：北京五环内，距离地铁站500米范围内有多少住宅小区？用热力图展示

AI执行：
1. 获取北京五环边界
2. 获取地铁站POI数据
3. 生成500m缓冲区
4. 空间交集查询住宅小区
5. 生成核密度热力图
6. 渲染到Cesium地图上
7. 输出分析报告
```

### 场景2：数据上传分析
```
用户：[上传 stations.geojson] 帮我分析这些站点的空间分布特征

AI执行：
1. 解析GeoJSON文件
2. 计算空间分布统计（质心、标准差椭圆、最近邻指数）
3. 识别聚类模式
4. 在地图上标注聚类区域
5. 生成分析报告
```

### 场景3：自然语言地图操控
```
用户：飞到上海陆家嘴，倾斜45度俯瞰，加载卫星底图

AI执行：
1. 地名→坐标转换（geocoding）
2. camera.flyTo(陆家嘴坐标, pitch: -45°)
3. 切换天地图影像底图
```

### 场景4：轨迹动画生成
```
用户：[上传 track.csv] 播放这个GPS轨迹的运动动画，速度10倍

AI执行：
1. 解析CSV（lon, lat, timestamp）
2. 生成SampledPositionProperty
3. 创建Entity + Path
4. 设置Clock multiplier=10
5. 播放轨迹动画
```

### 场景5：对比分析
```
用户：对比2020年和2025年北京城区绿地面积变化

AI执行：
1. 获取两期用地数据
2. 提取绿地图斑
3. 叠加分析（union/difference）
4. 分屏展示变化区域
5. 统计增减面积并生成报告
```

## 4. 能力定位

### 4.1 能力层级

| 层级 | 能力 | 权重 | 说明 |
|------|------|------|------|
| **L1 核心** | AI 对话式数据分析 | 40% | LangGraph Agent + 工具编排 |
| **L1 核心** | 矢量数据处理与空间分析 | 30% | 缓冲/查询/叠加/统计/geocoding |
| **L2 基础** | 2D 交互式可视化 | 20% | GeoJSON 图层/热力图/聚类/图表 |
| **L3 扩展** | 3D 数据展示 | 10% | 加载已有 terrain/3DTiles/影像服务（仅展示，不处理原始数据） |

### 4.2 能力边界

**能做：**
- 矢量空间数据全链路分析（GeoJSON/SHP/CSV/Excel/KML/KMZ/GPX）
- OSM / 天地图数据获取与分析
- 空间分析（缓冲区/空间查询/叠加/距离/统计/聚类）
- 地理编码 / 坐标转换
- 2D 交互式可视化（图层/热力图/聚类/轨迹动画）
- 加载已有 3D 服务（terrain URL / 3DTiles URL / 影像瓦片服务）

**不能做：**
- 地形 DEM 切片（需 C++ cesium-terrain-builder / CesiumLab）
- 倾斜摄影 OSGB → 3DTiles 转换（需 ContextCapture / CesiumLab）
- 大规模遥感影像处理（>500MB GeoTIFF 切片）
- BIM 模型转换（IFC → 3DTiles）
- 点云处理（LAS/LAZ → 3DTiles）

> 3D 数据 GeoAgent 的策略是"**不处理，但能加载**"：用户用第三方工具处理后，GeoAgent 加载已有服务 URL 进行展示。

## 5. 产品矩阵

| 产品 | 定位 | 技术栈 | 发布形式 |
|------|------|--------|---------|
| **GeoAgent** | AI 驱动地理空间分析平台（主产品） | Python + Vue3 + Cesium | GitHub 开源 |
| **cesium-mcp-dev** | Cesium 开发时 AI 助手 | TypeScript | npm 包 + GitHub |
| **cesium-mcp-runtime** | AI Agent 运行时操控 Cesium | TypeScript + WebSocket | npm 包 + GitHub |

三个项目独立开源，cesium-mcp 系列填补市场空白（Cesium 官方正在征集 MCP 方案但尚无产品）。
详见 `08-Cesium-MCP产品规划.md`。

## 7. 技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + TypeScript + Cesium | 地图交互与可视化 |
| UI组件 | Element Plus / Naive UI | 对话面板、结果展示 |
| 后端 | Python + FastAPI | API服务 |
| AI引擎 | LangChain / LangGraph | Agent编排、工具调度 |
| LLM | 通义千问 / DeepSeek / GPT-4 | 多模型支持 |
| 空间计算 | Turf.js (前端) + Shapely/GeoPandas (后端) | 空间分析 |
| 数据库 | PostGIS | 空间数据存储与查询 |
| 向量库 | ChromaDB / Milvus | GIS知识RAG检索 |
| 缓存 | Redis | 会话状态、结果缓存 |
| 部署 | Docker Compose | 一键启动 |

## 9. 差异化优势

| 对比维度 | 传统GIS平台 | 现有AI工具(ChatGPT等) | **GeoAgent** |
|----------|------------|----------------------|-------------|
| 空间分析 | ✅ 专业 | ❌ 只能说不能做 | ✅ AI驱动自动执行 |
| 上手门槛 | 高（需培训） | 低 | **极低（自然语言）** |
| 可视化 | ✅ | ❌ 只能出图片 | ✅ **交互式地图** |
| 自动化 | 需要编程 | 不能操作地图 | **全链路自动** |
| 开源 | 少 | 闭源 | **完全开源** |
