# 数据库设计

## 1. 数据库选型

| 数据库 | 用途 | 必要性 |
|--------|------|--------|
| **PostgreSQL + PostGIS** | 会话/消息存储 + 空间数据存储与查询 | MVP必须 |
| **Redis** | 缓存、会话状态 | MVP可选 |
| **ChromaDB** | GIS知识向量检索（RAG） | P2 |

### MVP阶段策略
- **直接使用 PostgreSQL + PostGIS** 作为唯一数据库，同时承担业务数据和空间数据存储
- 使用 **GeoAlchemy2** 作为 ORM 层，结合 SQLAlchemy 操作空间数据
- 通过 **Docker Compose** 一键启动 PostGIS 容器（`postgis/postgis:16-3.4`），零手动配置
- 空间分析优先使用 PostGIS 原生函数（ST_Buffer、ST_Within等），复杂分析降级到 Shapely/GeoPandas
- 上传文件存储在本地文件系统

> **为什么不用 SQLite 过渡？**
> 1. 避免 SQLite → PostgreSQL 的迁移成本和重复工作
> 2. 从第一天就拥有空间索引（GIST）和空间查询能力
> 3. PostGIS 是简历亮点，面试中可直接展示
> 4. Docker 部署极其简单：`docker run -d postgis/postgis:16-3.4`

### 2.2 表结构

#### sessions 会话表

```sql
CREATE TABLE sessions (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title       TEXT DEFAULT '新对话',       -- 会话标题（自动从第一条消息提取）
    model       TEXT DEFAULT 'deepseek-chat',
    status      TEXT DEFAULT 'active',      -- active | archived
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    updated_at  TIMESTAMPTZ DEFAULT NOW()
);
```

#### messages 消息表

```sql
CREATE TABLE messages (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id      UUID NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    role            TEXT NOT NULL,          -- 'user' | 'assistant'
    content         TEXT NOT NULL,          -- 消息文本内容
    thinking_steps  JSONB,                  -- [{step, total, content, status}]
    map_commands    JSONB,                  -- [{action, params}]
    charts          JSONB,                  -- [{chartType, title, ...}]
    report          TEXT,                   -- Markdown报告
    tokens_used     INTEGER DEFAULT 0,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_messages_session ON messages(session_id);
```

#### layers 图层表

```sql
CREATE TABLE layers (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id      UUID NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    name            TEXT NOT NULL,
    type            TEXT NOT NULL,          -- 'geojson' | 'heatmap' | 'tileset' | 'track'
    geometry_type   TEXT,                   -- 'Point' | 'LineString' | 'Polygon' | 'Mixed'
    feature_count   INTEGER DEFAULT 0,
    bbox            BOX2D,                  -- PostGIS 2D边界框
    style           JSONB,                  -- {color, opacity, size, ...}
    visible         BOOLEAN DEFAULT TRUE,
    data_ref        TEXT,                   -- 数据引用（文件路径或内联JSON标识）
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_layers_session ON layers(session_id);
```

#### uploaded_files 上传文件表

```sql
CREATE TABLE uploaded_files (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id      UUID REFERENCES sessions(id) ON DELETE SET NULL,
    filename        TEXT NOT NULL,
    file_path       TEXT NOT NULL,          -- 服务器存储路径
    file_type       TEXT NOT NULL,          -- 'geojson' | 'csv' | 'kml' | 'gpx' | 'shp'
    file_size       INTEGER NOT NULL,       -- 字节
    feature_count   INTEGER,
    geometry_type   TEXT,
    bbox            BOX2D,                  -- PostGIS 2D边界框
    crs             TEXT DEFAULT 'EPSG:4326',
    properties      JSONB,                  -- [{name, type, sample}]
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    expires_at      TIMESTAMPTZ             -- 自动清理时间
);
```

## 3. PostGIS空间数据能力

从 MVP 阶段起即使用 PostgreSQL + PostGIS，拥有完整的空间数据存储与查询能力。

### 3.1 空间数据表

```sql
-- 启用PostGIS扩展
CREATE EXTENSION IF NOT EXISTS postgis;

-- 空间数据表（存储用户上传和分析过程中产生的空间要素）
CREATE TABLE spatial_data (
    id              SERIAL PRIMARY KEY,
    session_id      UUID REFERENCES sessions(id) ON DELETE CASCADE,
    layer_id        UUID REFERENCES layers(id) ON DELETE CASCADE,
    name            TEXT,
    geom            GEOMETRY,              -- PostGIS几何类型
    properties      JSONB,                 -- 属性数据
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 空间索引
CREATE INDEX idx_spatial_data_geom ON spatial_data USING GIST(geom);
CREATE INDEX idx_spatial_data_session ON spatial_data(session_id);
CREATE INDEX idx_spatial_data_props ON spatial_data USING GIN(properties);
```

### 3.2 PostGIS空间分析函数

直接在数据库层完成空间分析，比应用层更高效：

```sql
-- 缓冲区分析
SELECT ST_Buffer(geom::geography, 500)::geometry AS buffer_geom
FROM spatial_data
WHERE layer_id = 'subway_stations';

-- 空间查询（在缓冲区内的点）
SELECT a.*
FROM spatial_data a, spatial_data b
WHERE b.layer_id = 'subway_buffer'
AND ST_Within(a.geom, b.geom)
AND a.layer_id = 'residential';

-- 距离计算
SELECT name, 
       ST_Distance(geom::geography, ST_SetSRID(ST_MakePoint(116.4, 39.9), 4326)::geography) AS distance_m
FROM spatial_data
WHERE layer_id = 'poi_data'
ORDER BY distance_m
LIMIT 10;

-- 核密度（需要PostGIS + pgRouting或应用层计算）
-- 聚类
SELECT ST_ClusterDBSCAN(geom, eps := 0.005, minpoints := 5) OVER() AS cluster_id, *
FROM spatial_data
WHERE layer_id = 'subway_stations';
```

## 6. 会话数据缓存层（Level 1）

数据不全部入库 PostGIS，工具间的空间数据通过**会话级内存缓存**传递，减少序列化/IO开销。详见 `02-技术架构设计.md § 7. 数据存储分层策略`。

### 6.1 SessionDataContext

```python
# backend/app/services/session_context.py
from dataclasses import dataclass, field
from typing import Dict, Optional
import time, uuid, sys

@dataclass
class DataRef:
    """会话中的一个数据引用"""
    ref_id: str                    # 唯一标识，如 "ref_subway_stations_a3f2"
    data: dict                     # GeoJSON FeatureCollection
    source: str                    # 来源工具名
    created_at: float              # 时间戳
    feature_count: int
    geometry_type: str             # Point/LineString/Polygon/Multi*
    bbox: list                     # [minLon, minLat, maxLon, maxLat]
    properties: list               # 属性字段名列表
    size_bytes: int                # 估算内存占用
    storage_level: int = 1         # 1=内存, 2=已升级到PostGIS
    postgis_table: Optional[str] = None  # Level 2时的表名/记录ID

class SessionDataContext:
    """每个会话一个实例，管理该会话所有的 data_ref"""
    
    def __init__(self, session_id: str):
        self.session_id = session_id
        self.data_refs: Dict[str, DataRef] = {}
        self.max_total_bytes = 100 * 1024 * 1024  # 100MB总上限
    
    def register(self, geojson: dict, source: str) -> str:
        """注册数据，返回 ref_id"""
        ref_id = f"ref_{source}_{uuid.uuid4().hex[:8]}"
        features = geojson.get("features", [])
        size = sys.getsizeof(str(geojson))  # 粗估
        
        ref = DataRef(
            ref_id=ref_id,
            data=geojson,
            source=source,
            created_at=time.time(),
            feature_count=len(features),
            geometry_type=self._detect_geom_type(features),
            bbox=self._calc_bbox(features),
            properties=self._extract_properties(features),
            size_bytes=size
        )
        
        # 超阈值自动升级到PostGIS
        if size > 10 * 1024 * 1024:
            self._upgrade_to_postgis(ref)
        
        self.data_refs[ref_id] = ref
        self._check_total_limit()
        return ref_id
    
    def get(self, ref_id: str) -> Optional[dict]:
        """获取数据，Level 2时从PostGIS加载"""
        ref = self.data_refs.get(ref_id)
        if not ref:
            return None
        if ref.storage_level == 2:
            return self._load_from_postgis(ref)
        return ref.data
    
    def summary(self, ref_id: str) -> Optional[dict]:
        """返回摘要信息（给Agent看的，不含全量数据）"""
        ref = self.data_refs.get(ref_id)
        if not ref:
            return None
        return {
            "data_ref_id": ref.ref_id,
            "source": ref.source,
            "feature_count": ref.feature_count,
            "geometry_type": ref.geometry_type,
            "bbox": ref.bbox,
            "properties": ref.properties,
            "storage_level": ref.storage_level
        }
    
    def list_refs(self) -> list:
        """列出所有 data_ref 摘要"""
        return [self.summary(rid) for rid in self.data_refs]
```

### 6.2 工具间数据流

```
工具A执行完毕
    │
    ├─ 输出 GeoJSON → session_context.register() → 得到 ref_id
    │
    ├─ 返回给 Agent: { data_ref_id, feature_count, geometry_type, bbox, ... }
    │                  (摘要，不含全量数据)
    │
    ▼
Agent 决策下一步
    │
    ├─ 调用工具B: buffer_analysis(data_ref="ref_xxx", distance=500)
    │
    ▼
工具B执行
    │
    ├─ actual_data = session_context.get("ref_xxx")  ← 从缓存取
    ├─ 计算缓冲区
    ├─ session_context.register(result) → 新 ref_id
    └─ 返回摘要
```

### 6.3 与 PostGIS 的协作

| 操作 | Level 1 (缓存) | Level 2 (PostGIS) |
|------|----------------|-------------------|
| 读取速度 | 纳秒级（内存直接引用） | 毫秒级（网络IO + 反序列化） |
| 空间查询 | Shapely 内存计算 | ST_Within 等走GIST索引 |
| 适用数据量 | < 10MB / < 50000要素 | 不限 |
| 持久化 | 否（进程退出即丢失） | 是 |
| 跨会话 | 否 | 是 |

**何时使用PostGIS空间函数而非内存计算：**
- 数据量 > 50000 要素（内存Shapely变慢）
- 需要空间索引加速的反复查询（同一数据集多次ST_Within）
- 需要持久化的最终分析结果

### 6.4 清理策略

| 触发条件 | 动作 |
|----------|------|
| 会话关闭/超时 | 清除该会话所有 Level 1 缓存 |
| 单个ref > 10MB | 自动升级到 PostGIS，内存中仅保留摘要 |
| 会话总缓存 > 100MB | LRU淘汰最早的 ref（先检查是否需升级） |
| 用户说"保存这个结果" | 标记为最终图层，升级到 PostGIS + layers 表 |

### 7.2 SQLAlchemy + GeoAlchemy2 初始化

```python
# backend/app/database.py
from sqlalchemy import create_engine, text
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker, DeclarativeBase
from geoalchemy2 import Geometry
from app.config import settings

# 异步引擎
engine = create_async_engine(
    settings.DATABASE_URL,  # "postgresql+asyncpg://geoagent:xxx@localhost:5432/geoagent"
    echo=settings.DEBUG,
)

AsyncSessionLocal = sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

class Base(DeclarativeBase):
    pass

async def init_db():
    """创建所有表 + 启用 PostGIS 扩展"""
    async with engine.begin() as conn:
        await conn.execute(text("CREATE EXTENSION IF NOT EXISTS postgis"))
        await conn.run_sync(Base.metadata.create_all)

async def get_db():
    """FastAPI 依赖注入"""
    async with AsyncSessionLocal() as session:
        yield session
```

### 6.3 ORM 模型示例

```python
# backend/app/models/session.py
import uuid
from datetime import datetime
from sqlalchemy import Column, Text, Integer, Boolean, DateTime, ForeignKey
from sqlalchemy.dialects.postgresql import UUID, JSONB
from geoalchemy2 import Geometry
from app.database import Base

class Session(Base):
    __tablename__ = "sessions"
    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    title = Column(Text, default="新对话")
    model = Column(Text, default="deepseek-chat")
    status = Column(Text, default="active")
    created_at = Column(DateTime(timezone=True), default=datetime.now)
    updated_at = Column(DateTime(timezone=True), default=datetime.now, onupdate=datetime.now)

class Message(Base):
    __tablename__ = "messages"
    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    session_id = Column(UUID(as_uuid=True), ForeignKey("sessions.id", ondelete="CASCADE"), nullable=False)
    role = Column(Text, nullable=False)
    content = Column(Text, nullable=False)
    thinking_steps = Column(JSONB)
    map_commands = Column(JSONB)
    charts = Column(JSONB)
    report = Column(Text)
    tokens_used = Column(Integer, default=0)
    created_at = Column(DateTime(timezone=True), default=datetime.now)

class SpatialData(Base):
    __tablename__ = "spatial_data"
    id = Column(Integer, primary_key=True, autoincrement=True)
    session_id = Column(UUID(as_uuid=True), ForeignKey("sessions.id", ondelete="CASCADE"))
    layer_id = Column(UUID(as_uuid=True), ForeignKey("layers.id", ondelete="CASCADE"))
    name = Column(Text)
    geom = Column(Geometry(srid=4326))     # PostGIS 几何列
    properties = Column(JSONB)
    created_at = Column(DateTime(timezone=True), default=datetime.now)
```
